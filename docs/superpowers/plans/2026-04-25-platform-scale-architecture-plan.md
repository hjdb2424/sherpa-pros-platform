# Platform Scale Architecture — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate Sherpa Pros from single-region MVP to multi-region, SOC 2 Type 2 certified, 99.95% SLA platform capable of 1M+ MAU. Execution window: Q3 2026 through Q2 2028 (Type 2 issuance). 7 parallel work streams totaling ~50 bite-sized tasks.

**Source spec:** `docs/superpowers/specs/2026-04-25-platform-scale-architecture-design.md`

**Owners:**
- **P** = Phyrom (founder, decision authority on every vendor + budget)
- **CTO** = future hire (Phase 4A); until then, Phyrom acts in this role
- **SRE** = future hire (Phase 4A); until then, Phyrom + part-time advisor
- **DBA** = future hire (Phase 4A); until then, Neon support + Postgres advisor
- **Security** = Vanta + auditor + pen test firm
- **AI** = Claude sub-agent dispatch (parallelized in waves)

**Critical-path dependencies:**
- WS1 (SOC 2 Type 1) gates WS7 step 5 (Canadian region — enterprise PM customers require Type 1)
- WS2 (Observability) gates WS3 step 4 (sharding rollout — need APM to detect regression)
- WS4 (WAF + rate limiting) gates any high-profile public launch
- WS5 (SRE practices) gates WS3 + WS7 (multi-region work without on-call discipline = burnout)
